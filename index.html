About This FAQ

This FAQ was created by the various members of the Discord C# Channel to help with anyone new to C# and looking for guidance. There are many, many different approaches to learning the .NET stack, none of which could be described as wrong. This FAQ aims to inform about the most common recommendations, advice, tools, frameworks etc. If you're not looking for advice and you're just here for the links to resources, click here.

Where Do I Start?

That largely depends on where you want to end. The sections below give some advice depending on if your priority is mobile development, web development, game development or just-get-me-a-job development. That said, starting with vanilla C# is a given.

Starting Development

Installing Visual Studio is (by a huge margin) the most recommended first step. Visual Studio Community Edition is free and can be downloaded here. Visual Studio is highly regarded and you'll find that lots of learning materials will assume you're using it.

The next step depends heavily on your current skill level with programming in general and the best option is usually grabbing a book on C# and diving in at the point you feel comfortable. The following are some books that our users have had some success with (most links are to Amazon but with a little skillful googling you should find them floating around).

  • The C# Yellow Book by Rob Miles, University of Hull - A very accessible book with the added bonus of being free

  • C# In Depth by Jon Skeet

  • Head First C# by O'Reilly

  • Beginning C Sharp Programming with Visual Studio by Perkins, Hammer, & Reid - This book also covers installing Visual Studio if you need guidance

  • A list of free programming books, compiled on GitHub

If you want to follow some online tutorials here are some recommended links

  • Microsoft Developer Network

  • Tutorials Point C#

  • LearnCS.org

  • CSharp Station

Learning about general programming concepts that aren't specific to C# is a good plan. Most people you ask about where to start will often tell you to begin with design patterns, data structures, algorithms etc. This might be less hands on and a little drier than you want, but learning them sooner can save sweat and tears later. Below are some recommended books about programming in general.

  • Code Complete, Second Edition, by Steve McConnell

  • Head First Design Patterns by Bates, Freeman, Robson, & Sierra

  • The Pragmatic Programmer by Andrew Hunt & David Thomas

  • Clean Code by Robert C. Martin

  • Introduction to Algorithms, by CLRS

General C# Development

Most people viewing this page likely want to move into the software development industry. So what makes you a good candidate to hire? Well, a degree in Computer Science is a good start. Other than that you can practise development using tools and design patterns that software development houses use themselves, then use these skills to develop a portoflio of applications on GitHub and also contribute to Open Source projects.

Companies developing desktop applications use WinForms, WPF, and UWP for their GUIs. WinForms is the oldest framework but you won't have to look hard to find businesses maintaining software using this framework. WPF is newer and more commonly listed on job requirements. UWP is the newest framework to allow development for applications on Windows 10 and Windows Phones/Tablets. If you have plenty of time then learn WinForms, then WPF, then UWP. If you don't, start with WPF. If you need more info on WinForms and WPF that you don't find in the books linked over this page then Google is your friend, there are lots of resources out there for these frameworks.

If you are studying WPF it is recommended to learn the MVVM pattern sooner rather than later. A good approach would be to learn WPF until you understand data bindings, then jump over to learning MVVM. There are not as many resources for learning MVVM as there should be, but a few small guides are listed below.

  • MSDN Patterns - WPF Apps With The MVVM Design Pattern

  • TutorialsPoint MVVM

  • CodeProject WPF MVVM step by step

  • MVVM Simplified

  • RArcher - Embracing MVVM (Context of Windows 8 Store Development)

Other tools used in a business setting that you should learn are the various version control systems (with Git being the most common), the various testing approaches (with Unit Testing being a good place to start), and a good understanding of LINQ will be very helpful. The sooner you start using these the better.

Game Development

Game Development in the .NET framework is mostly done within the Unity framework, which you can install from here. Another popular framework is MonoGame which allows cross platform development. Below is a list of some resources to get you started.

  • Tutorials on the Unity3d Website

  • MSDN Unity: Developing Your First Game with Unity and C#

  • GamesFromScratch Monogame Tutorial Series

  • RB Whitakers Monogame Tutorials

Mobile Development

In the .NET framework, Xamarin allows developers to write code that can be deployed to multiple mobile platforms. In March 2016, Microsoft made Xamarin free (and then later Open Source), which means opportunities to get paid to develop with it are only likely to increase.

You can install Xamarin into Visual Studio from here.

Web Development

For web development you will want to study ASP.Net and the MVC design pattern which is very common. Microsofts ASP.NET MVC Tutorials are a great place to start for this. You will also want to learn the Entity Framework and SQL at which point you'll have covered the entire .NET stack. After these you should gain some knowledge in HTML, CSS, Javascript, and the various frontend frameworks (Angular.js, bootstrap etc) which we won't cover here.

Free Student Resources

If you're a student you can get a bunch of free stuff. The best place to start in this case is with the Microsoft Dreamspark program. This will give you free copies of software, a free 90 day subscription to Pluralsight, more free resources from GitHub, and quite a lot of other stuff. You can sign up for it here.
